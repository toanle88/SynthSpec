# Contributing to SynthSpec

Thank you for your interest in contributing to SynthSpec! We welcome community contributions to help improve the tool.

## How to Contribute

To keep the codebase maintainable and robust, we ask that all contributors follow our guidelines:

1. **Read the Full Contributing Guide**: Detailed guidelines for setup, architecture principles, code style, testing, and PR workflow can be found in our comprehensive **[Contributing Guide (docs/development/contributing.md)](docs/development/contributing.md)**.
2. **Review Coding Standards**: Check the [Go Coding Standards (docs/development/coding-go.md)](docs/development/coding-go.md) and [General Coding Standards (docs/development/coding.md)](docs/development/coding.md).
3. **Follow the Branching Model**: Create feature branches from `main` using standard naming conventions:
   - `feat/feature-description`
   - `fix/bug-description`
   - `docs/doc-description`
4. **Ensure Test Coverage**: All packages must have unit tests, and new code must be fully tested before merging. Make sure no real network calls are made in the tests.

For any questions or troubleshooting help, refer to the [Maintainer Runbook (docs/operations/runbooks.md)](docs/operations/runbooks.md).
